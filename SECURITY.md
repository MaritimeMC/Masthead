# Security Policy

## Supported Versions

Only current production releases and branches to be merged into master will have
vulnerabilities addressed by Senior Developers. It is the job of the original developer to 
discover and remove any non-production vulnerabilities.

| Version | Supported          |
| ------- | ------------------ |
| Latest production release   | :white_check_mark: |
| Pull requests to master   | :white_check_mark:                |
| Development branches / test networks   | :x: |

## Reporting a Vulnerability

Vulnerabilities within MastheadControl should be reported _securely_ to a Senior Developer
via our main platform of communication. This should be reported privately and **not** in a 
public room/chat.
