# MastheadControl
Overhead control system for the Masthead suite.

Commit messages should follow [Conventional Commits](https://conventionalcommits.org)

To request a feature when this plugin is in production, [create an issue](https://github.com/Minedroid-Network/MastheadControl/issues/new/choose).
<br>Bugs should be reported using the **Bug report** issue template.

The progress of issues, pull requests, and features can be tracked using [our tracker project](https://github.com/Minedroid-Network/MastheadControl/projects/1). 
This automatically updates when an issue / pr status is changed.

We make use of the GitHub branch workflow:
* Anything in the _master_ branch should always be deployable. 
* A long-term development project before release into production should be developed on the _develop_ branch.
* A pull request should be created before deployment to merge into master.
* Branches for specific features / bug-fixes should have descriptive names.

We also make use of GitHub Milestones:
* Issues and pull requests can be associated with certain goals / deadlines using milestones.
